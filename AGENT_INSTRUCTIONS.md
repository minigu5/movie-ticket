# 📋[대구과학고등학교 '영화대교' 예매 시스템 프로젝트 명세서 V3]

## 1. 프로젝트 개요
*   **개발자/기획자:** 대구과학고 2208 신민규 (동아리 '영화대교')
*   **목적:** 학교 강당에 전교생 및 교직원을 위한 무료 영화관 구축 시스템
*   **기술 스택:** Next.js (App Router, React), Tailwind CSS, Supabase (PostgreSQL), Vercel 호스팅, Nodemailer (Gmail 발송)
*   **최근 주요 변경 사항:** 결제 및 팝콘 기능을 완전히 제거하고 **100% 무료 순수 예매 시스템**으로 개편됨.

## 2. 데이터베이스 구조 (Supabase)
1.  **`reservations` (예매 내역):** 예매자 학번(`student_id`), 이름(`student_name`), 선택 좌석(`seat_number`), 결제 상태(항상 `confirmed`), 현장 키오스크 출력 여부(`is_printed`). *(팝콘 컬럼이 남아있다면 'none'으로 무시됨)*
2.  **`movie_settings` (영화 설정):** 영화 제목, 상영 일시(`date_string`), DB 필터링용 날짜(`db_date`), 장소, 포스터 URL, 예매 마감시간, 관람가 등급(`age_rating` - 추가됨), 장소별 VIP 좌석 범위(행/열 기준).
3.  **`blacklist` (블랙리스트):** 좌석 미정리 학생 등을 등록하여 예매 영구 차단.
4.  **`student_auth` (영구 비밀번호):** 고유 식별자(`student_id`)와 4자리 숫자 비밀번호 영구 보관. 이메일 재설정용 토큰 보관.
5.  **`activity_logs` (감사 로그):** 예매, 취소, 블랙리스트, 발권, 비밀번호 재설정 등 모든 활동 기록.

## 3. 핵심 비즈니스 로직 및 예외 처리 사항 (에이전트 필독 🚨)
### A. 사용자 인증 및 교직원 예외 처리
*   **학생:** 4자리 학번을 입력하며, 명렬표 데이터(`STUDENT_LIST`)와 이름이 정확히 일치해야 함.
*   **교직원:** 학번 입력란에 "교직원"이라고 입력. `STAFF_LIST`와 일치해야 함.
*   **인증 키(Auth Key) 분리 (중요):** 교직원은 학번이 모두 "교직원"이므로 덮어쓰기 버그를 방지하기 위해, `student_auth` 테이블 및 예매/취소 검증 시 **학생은 '학번', 교직원은 '이름'을 고유 ID(`authKey`)로 사용**함.

### B. 좌석 시스템 및 UI 디자인
*   **좌석 번호 알고리즘 (블록형):**
    *   중강당: A구역(왼쪽 7칸), B구역(오른쪽 7칸). `A01~A63`, `B01~B62`.
    *   대강당: A구역(왼쪽 9칸), B구역(중앙 9칸), C구역(오른쪽 9칸). `A001~A162` 방식.
*   **동아리(VIP) 전용 좌석:** DB 설정값을 기반으로 `useMemo`를 통해 `vipSeats` Set을 미리 계산. 동아리원 명단(`CLUB_MEMBERS`)에 없는 사람이 클릭 시 예매 차단. (TDZ 에러 방지를 위해 `getSeatId` 함수가 `useMemo`보다 위에 선언되어 있음).
*   **모바일 UX:** 좌석 버튼의 가로 너비는 좁히고 글자를 키움(`whitespace-nowrap` 적용). 화면 스크롤 시 스크린(SCREEN) 그래픽이 중앙(B구역)에 정렬되어 있으며, 모바일에서는 좌측 상단에 "옆으로 밀어서 확인 👉" 애니메이션이 동작함.

### C. 예매 프로세스
*   팝콘 선택 및 결제 로직이 삭제됨.
*   좌석 클릭 -> 학번/이름/비밀번호 입력 -> 즉시 `confirmed` 상태로 DB 저장 및 안내 메일 발송.
*   자리 이동: 이미 예매한 사용자가 다른 빈 좌석을 클릭하고 예매를 진행하면, 기존 자리가 취소되고 새 자리로 자동 이동.

## 4. 관리자 페이지 (`/admin`)
*   **대시보드:** 예매 내역 열람, 예약 강제 취소, 최근 100건 활동 로그, 블랙리스트 관리, 영화 설정 변경(관람가 등급 포함).
*   **대량 홍보 메일:** 1~3학년, 교직원, 동아리 등 그룹을 선택해 홍보 메일 발송. Vercel 타임아웃을 막기 위해 15명씩 묶어서(Chunking) 발송.
*   **발권 초기화 기능:** 영수증 출력 중 오류가 발생한 학생을 위해, 관리자가 클릭 한 번으로 해당 학생의 티켓 `is_printed` 상태를 `false`로 돌릴 수 있는 버튼 존재.

## 5. 현장 키오스크 발권기 (`/print`)
*   **관리자 잠금:** 현장 태블릿 오작동 방지를 위해 최초 진입 시 비밀번호("영화대교최고")를 입력해야 발권 화면이 열림.
*   **출력 제어:** `is_printed`를 확인하여 1인 1매만 출력. 출력 후 브라우저 상태 초기화. 네비게이션 버튼 없음(탈출 불가).
*   **감열식 프린터 디자인:** `@media print`와 `@page { margin: 5mm; }` 적용. 흐리게 나오는 현상을 막기 위해 굵은 글꼴과 테두리 박스 사용. 바코드는 무조건 1줄로 표시.
